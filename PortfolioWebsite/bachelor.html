Subject Description

The bachelor thesis is based on either a problem/thesis proposal obtained from business, public sector or the academic community. The proposals must be pre-approved by the course coordinator. Through the assignment, students will identify, formulate and solve relevant problems within programmers' field of work. The candidate must use knowledge and skills from several subject areas in the study, as well as independent specialization where necessary. Through the bachelor thesis, the candidate must contribute to either research or development. The candidate must also acquire skills in project management, planning, implementation and documentation of a project work. The bachelor thesis must include software development, and may also have elements of research, innovation or entrepreneurship.

What the Bachelor for was about

Me and my group got a task from Innoveria with the the following task from them

Our task is to create a web application that collects, stores, and displays electrical usages from sensors placed on machines within a factory setting. The application should be available on PC and tablet and deliver an intuitive user experience. It should also be scalable and expandable, allowing new features to be added with relative ease by a new developer.

Additionally, the application should contain the following functionality:

  • Overview page that displays collected data:
    • Graph view
    • Table view
    • Layered sorting system, layered into:
      • Building
      • Department (optional)
      • Production line
      • Machine
    • Filtering results by layer, individual machine, highest consumer, and/or time of data
  • Access rights
  • Page for ENØK measures:
    • Accepted/proposed measures
    • An ENØK superuser with permissions to accept/discard proposed measures
  • Adding and removing sensors
  • Adding and removing gateways

Technologies Used:

  • Docker and Docker compose
  • ShadCN
  • Chirpstack
  • Vite
  • Vutility HotDrop
  • WisGate Edge Lite 2

Programming Languages Used:

  • React.js
  • Golang
  • MySQL

The Project

The project was divided into three main components: Frontend (design), Backend (APIs and Database), and the LoRaWAN server (connection and data collection).

Throughout the project, the team utilized Scrum as an agile framework.

The project was successful, and you can view the final product on the right. It was built using the technologies and programming languages mentioned above. This system integrates the database, the website, and Rak gateways through APIs, automatically collecting, processing, and storing data from sensors monitoring the power usage of manufacturing machines. Users can view this data in near real-time.
